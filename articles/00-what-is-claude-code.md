# Claude Code 到底是什么？一个 AI 编程助手的完整拆解

> Anthropic 的 Claude Code 源码因 npm 打包事故完整泄露。我花两周通读了全部源码，这篇文章用最直白的方式告诉你：Claude Code 究竟是一个怎样的系统，它和你想象中的"AI 写代码"有什么不同。

---

## 不只是"帮你写代码"

大部分人对 AI 编程助手的印象停留在"给它一段描述，它帮你写代码"。如果 Claude Code 只做这件事，它不需要 1,900 个源文件。

Claude Code 是一个**住在你终端里的 AI 同事**。它能读你的代码、改你的文件、跑你的命令、记住你的偏好、接入你的工具——而且它知道什么时候该问你"这个操作我能执行吗？"

具体来说，它能做这些事：

| 能力 | 例子 |
|------|------|
| 读代码 | 浏览项目结构，搜索特定函数，理解文件间依赖 |
| 写代码 | 创建新文件，编辑现有代码，精确到行级的修改 |
| 跑命令 | 执行 Shell 命令，运行测试，管理 Git 操作 |
| 多步推理 | "找到这个 bug → 分析原因 → 修改代码 → 跑测试验证" 一条龙 |
| 团队协作 | 多个 AI Agent 分工——一个负责规划，多个负责执行 |
| 记住你 | 记住项目约定、你的偏好，下次对话自动沿用 |
| 接外部工具 | 通过 MCP 协议接入数据库、API、浏览器等任何工具 |
| 主动行动 | KAIROS 模式下监控项目变化，不等你提问就主动处理 |

但真正有意思的不是"它能做什么"，而是"它怎么做到的"。

---

## 核心概念一：Agent Loop（推理循环）

传统的聊天机器人是"你问一句我答一句"。Claude Code 不是这样——它是一个**循环**。

你说"帮我修这个 bug"，它不会直接给你一段代码。它会：

1. 先去读相关文件，理解代码结构
2. 找到 bug 的位置，分析原因
3. 修改代码
4. 跑测试看看改对了没有
5. 如果测试没过，回到第 2 步重新分析
6. 测试通过了，才告诉你"搞定了，改了这些地方"

这个"思考 → 行动 → 观察 → 再思考"的循环，就是 Agent Loop。每一轮循环里，AI 会决定需要调用哪个工具（读文件？改代码？跑命令？），执行后拿到结果，再决定下一步。

一个简单的"修 bug"请求，可能在内部跑了 4-5 轮循环，调用了十几个工具。你看到的只是最终结果。

## 核心概念二：工具系统（40+ 种能力）

AI 光有"想法"不够，还需要"手"。Claude Code 内建了 40 多种工具，让它能真正操作你的电脑：

**基础操作**
- 读文件、写文件、精确编辑代码的某几行
- 搜索文件名（Glob）、搜索文件内容（Grep）
- 执行 Shell 命令（用来跑测试、安装依赖、Git 操作等）

**高级能力**
- 调用子 Agent（把复杂任务拆给"下属"）
- 通过 MCP 协议接入任何外部工具（数据库、API、浏览器……）
- Computer Use：直接看你的屏幕、操作你的鼠标键盘（macOS）

每个工具不是随便调的——每次调用都要过**权限检查**。AI 想跑一条 Shell 命令？先检查这条命令是不是安全的。想改一个文件？先看看这个路径是不是敏感路径。这就引出了下一个核心概念。

---

## 核心概念三：安全架构（7 层防护）

让 AI 操作你的电脑，最大的顾虑是"它会不会搞砸？"

Claude Code 对此的回答是**7 层安全防护**，层层嵌套，任何一层都能拦住危险操作：

**第 1 层：组织策略** — 你的公司管理员可以远程下发规则，比如"禁止删除生产数据库"

**第 2 层：权限模式** — 你自己选择的安全等级。最严格的模式下，每个操作都要你确认；最宽松的模式下，大部分操作自动通过——但核心安全检查依然强制执行

**第 3 层：声明式规则** — 你可以配置具体的"什么能做、什么不能做"。比如"允许 `git commit`，但 `git push --force` 必须问我"

**第 4 层：工具自检** — 每个工具有自己的安全逻辑。比如 Shell 工具会把命令解析成语法树，识别出 `rm -rf /` 这种危险操作，即使它被包裹在一堆无害的命令中间

**第 5 层：文件系统保护** — `.git/`、`.ssh/`、Shell 配置文件等敏感路径，无论什么模式都会要求确认

**第 6 层：沙箱** — 即使命令被允许执行了，也可以在一个隔离的沙箱里运行，限制它能访问的文件和网络

**第 7 层：信息安全** — 内部版本的功能代码在外部版本中被物理删除（不是隐藏，是不存在）

一个直觉类比：这像是银行的保险库——不是只有一道门，而是从大厅保安、身份验证、密码门禁、生物识别、摄像监控、金库锁、到保险柜，每一层都是独立的防线。

最重要的设计原则是：**即使你选了最宽松的模式，有些安全检查依然不能跳过**。"相信用户"和"保护用户"不矛盾。

## 核心概念四：记忆系统（它怎么记住你）

和 AI 聊天最痛苦的事是什么？每次开新对话，它什么都不记得。

Claude Code 有一套 **7 层记忆系统**来解决这个问题：

**你手动告诉它的记忆**（项目级配置文件）
- 可以写在项目的 `CLAUDE.md` 文件里，比如"这个项目用 Go 语言，测试用 pytest"
- 团队成员共享，提交到 Git 里
- 还有个人专用的本地版本，不会提交到代码库

**它自己学到的记忆**（AI 自动提取）
- 每次对话结束后，它会在后台把有价值的信息提取出来，存到记忆文件里
- 比如"用户偏好用函数式写法"、"这个项目的数据库迁移要用 alembic"
- 记忆文件像一本索引——标题页很短（始终加载），详细内容存在子文件里，需要时再查

**离线记忆整合**（AutoDream 系统）
- 这是最有意思的设计——它会在后台启动一个"做梦"的过程
- 像人类睡觉时整理白天记忆一样，把散落的信息归类、合并、精简
- 完全在后台运行，不影响你正常使用

## 核心概念五：上下文管理（在有限空间里做无限的事）

所有 AI 模型都有一个"上下文窗口"——它一次能看到的信息量有上限。Claude Code 要在这个有限的空间里驾驭可能几百轮的长对话，怎么做？

它的策略是**多级压缩**，从最轻量到最重量：

1. **折叠**：连续的只读操作（比如搜了 10 个文件）合并成一条摘要
2. **外溢**：工具返回的大量结果（比如一个超长文件的内容）不放在对话里，而是存到磁盘上，对话里只留一个引用指针
3. **摘要**：当对话真的太长了，用 AI 把前面的对话总结成一段摘要，替换掉原始消息
4. **熔断**：如果压缩本身也失败了（比如要压缩的内容本身就太长），停止尝试，防止死循环

你不需要关心这些——它全自动运行。效果就是：你可以和 Claude Code 聊几百轮，它不会"忘记"前面说了什么，也不会因为对话太长而崩溃。

## 核心概念六：可扩展性（不只是内建功能）

Claude Code 不是一个封闭系统。它有四种扩展方式：

**MCP 协议**：一种开放标准，让你可以用任何编程语言写一个"工具服务器"，Claude Code 自动发现和使用。比如你可以写一个连接公司内部数据库的 MCP 服务器，Claude Code 就能直接查数据库了。

**技能（Skills）**：用 Markdown 文件定义的"技能包"。放一个 `.md` 文件在项目目录下，就能教 Claude Code 新的工作流程。不需要写代码。

**插件（Plugins）**：更完整的扩展形式，可以添加命令、工具、Agent。通过 Marketplace 分发。

**Hooks**：在 AI 的各个操作节点上挂自定义逻辑。比如"每次 AI 要执行 Shell 命令前，先检查公司的安全策略"。

## 核心概念七：多种运行形态

Claude Code 不只活在终端里：

| 形态 | 使用场景 |
|------|---------|
| **交互式 CLI** | 在终端里和它对话，最常用的方式 |
| **SDK / Headless** | 被其他程序调用，没有 UI，通过 JSON 协议通信。IDE 插件（VS Code、JetBrains）就是这样用的 |
| **Server 模式** | 跑一个本地 HTTP 服务，IDE 通过网络连接 |
| **Remote 模式** | 本地终端连到云端容器执行，适合需要特定环境的场景 |
| **KAIROS 模式** | "始终在线"的主动助手，监控你的项目，不等你提问就主动行动 |

## 它和其他 AI 编程工具有什么不同？

看完源码后，我觉得 Claude Code 和市面上大部分 AI 编程工具的核心差异在三点：

**1. 它是一个完整的 Agent 系统，不是 API 套壳。** 很多工具本质上就是在调 API 前后加一点处理。Claude Code 有 1,900 个源文件、40 多个工具、7 层安全防护、7 层记忆系统——这是一个经过系统性工程设计的产品。

**2. 安全不是可选项。** 大部分 AI 工具的"安全"就是一个确认弹窗。Claude Code 用了 7 层防护，权限决策有 10 多个判定步骤，Shell 命令会被解析成语法树来检查安全性。而且核心安全检查无法被任何方式跳过。

**3. 它在为"AI 同事"这个愿景做准备。** 源码里有 30 多个未公开的功能开关——多 Agent 协作、主动监控、语音模式、深度规划（把复杂任务交给更强的模型思考 30 分钟）。这些指向的不是"更好的代码补全"，而是"一个能独立工作的 AI 团队成员"。
